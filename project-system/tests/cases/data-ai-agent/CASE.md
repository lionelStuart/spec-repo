# Demo Case: Data AI Agent

## Scenario

Create a repository memory system for a "Data AI Agent" product.

The product goal is to help analysts upload tabular data, inspect schema quality, ask natural-language questions, and generate charts with traceable execution steps.

## What This Case Tests

1. Can the skill bootstrap a usable repo structure for a real project?
2. Can it separate project context, spec, task, status, and decisions?
3. Can it preserve active progress and next steps without relying on chat memory?
4. Can it capture a reusable learning and promote it into a reusable skill?
5. Can it fit a domain-specific project instead of staying generic?

## Expected Outcome

The generated project memory repo should contain:

- repo-level instructions
- project goals and constraints
- active status and next steps
- a domain-specific architecture
- at least one spec
- at least one task
- at least one technical decision
- at least one learning
- at least one reusable skill derived from the learning

## Score Dimensions

- Structure completeness
- Progressive loading protocol
- Spec and task discipline
- Write-back quality
- Self-bootstrapping through learnings and skills
- Domain fidelity for a data AI agent
